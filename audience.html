Standing + Sitting Players Hero Layout

Topic: CSS Positioning for a Layered Hero

2-minute demo

What I Built

I created a hero section that looks like a team photo: 5 standing players behind and 4 sitting players in front, using separate transparent PNG images.

Artifact Slide 1 shows the final result.

Why Positioning Matters

  • Overlap control: absolute positioning makes layering predictable.
  • Stable anchor: a relative wrapper defines the coordinate system.
  • Front/back: z-index keeps sitting players in front.

Key Idea: Click a Class

HTML → CSS highlight

HTML (pick a class)

Use a relative container + absolute players + z-index to control overlap like a layered collage.

Click a class name below. The matching CSS rule will highlight on the right.

<div class="hero">
  <div class="players">
    <div class="player back">...</div>
    <div class="player front">...</div>
  </div>
</div>
relative = anchor (coordinates)
absolute = precise overlap
z-index = front/back

CSS (highlights)

.players {
  position: relative;
}

.player {
  position: absolute;
  bottom: 0;
}

/* sitting players in front */
.player.front:nth-child(9) {
  z-index: 7;
}

Key Takeaways

  • Transparent PNGs let you treat each player as a layer.
  • position: relative + position: absolute gives precise control.
  • z-index is the simplest way to control front/back.
